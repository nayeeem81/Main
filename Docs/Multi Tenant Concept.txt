## Multi-tenancy Architectural Model

### Concept
Multi-tenancy is an architectural model where a single instance of a software application and its underlying infrastructure serves multiple independent customers ("tenants"). While all tenants share the same server resources and code, their data is strictly isolated and hidden from one another. This model forms the backbone of modern cloud computing and Software-as-a-Service (SaaS) platforms.

### Key Characteristics 

1. Shared Resources: Infrastructure, application code, and databases are centralized, allowing the provider to maximize efficiency and lower costs.

2. Logical Isolation: Despite sharing resources, each tenant’s data is securely walled off. Tenant A cannot see or access Tenant B's data or configurations.

3. Centralized Management: Updates, patches, and feature rollouts are applied centrally by the provider, instantly benefiting all tenants.

4. Customizability: Tenants can often customize their specific workspace (e.g., logos, user interface colors, or workflow rules) without altering the core software.


### Common Use Cases

1. SaaS Platforms: Applications like Salesforce, HubSpot, or Google Workspace operate on multi-tenant architecture to serve millions of users efficiently.

2. Cloud Infrastructure: Platforms like AWS, Microsoft Azure, and Google Cloud often utilize multi-tenancy to provide isolated virtual environments to different clients within the same data centers.

### Multi-Tenant vs. Single-Tenant

Understanding the difference clarifies why businesses choose multi-tenancy:

Multi-Tenant: Shared environment. Highly cost-effective and easy to maintain, but offers less control over underlying hardware and configurations.

Single-Tenant: Dedicated environment. Each customer gets their own separate software instance and servers. This offers maximum security and customizability but is significantly more expensive and harder to maintain.

### Potential Challenges

1. The "Noisy Neighbor" Effect: Because resources are shared, one tenant experiencing a massive surge in traffic or processing load can potentially slow down the application for other tenants.

2. Customization Limits: Because everyone uses the same foundational code, tenants cannot modify the underlying software code to suit highly niche requirements.

For a deeper dive into how this architecture is implemented technically, you can explore the Kubernetes Multi-Tenancy Guide or read more about IBM's Software Multitenancy Definition.

### Multi-Tenant & Multi-Tenant Architecture

1. Software Architecture: In multi-tenant software architecture, also called software multitenancy. Multi-tenancy is an architecture in which a single instance of a software application serves multiple customers.

2. Database Architecture: Database Per Tenant or Shared? In a shared database approach for multi-tenant applications, multiple tenants utilize the same database instance and schema.   

### Multi-tenant ASP.NET Core 8.0 MVC application

To implement a multi-tenant ASP.NET Core 8.0 MVC application using a shared database (shared schema), you must isolate data using a tenant identifier column (TenantId) across your application tables, including ASP.NET Core Identity. Extend the default IdentityUser and IdentityRole to include the TenantId. This architecture uses EF Core Global Query Filters to automatically isolate data and custom middleware to resolve the tenant from the context. Create a scoped service to carry the active tenant information throughout an HTTP request lifecycle. Create a middleware component to extract the current tenant identifier from the request (e.g., from the sub-domain, route value, cookie, or header). Inherit from IdentityDbContext passing your custom ApplicationUser and ApplicationRole types. Override OnModelCreating to establish Global Query Filters and adjust constraints. Because standard ASP.NET Core Identity assumes unique usernames globally, you must relax user validation configuration so the same username or email can register across separate tenants. Register your services and middleware dependencies in the application pipeline. When creating or managing structural operations inside an MVC Controller, users and roles automatically align with the resolved tenant layer. Use standard [Authorize(Roles = "...")] filters seamlessly.

### Critical Development Notes: 
***
1. Initial Database Migrations: When running commands (dotnet ef migrations add), ensure the execution includes the initial structural layout generation containing the altered composite unique constraints over identity properties.

2. Finding Records Safely: Avoid accessing query objects using Find() or FindAsync() methods since EF Core bypasses Global Query Filters for direct primary-key matching. Use FirstOrDefaultAsync(x => x.Id == id) instead.

using Microsoft.AspNetCore.Identity;

public interface IMustHaveTenant
{
    public string TenantId { get; set; }
}

public class Tenant
{
    public string Id { get; set; } 
    public string Name { get; set; }
}

public class ApplicationUser : IdentityUser, IMustHaveTenant
{
    public string TenantId { get; set; }
}

public class ApplicationRole : IdentityRole, IMustHaveTenant
{
    public string TenantId { get; set; }
    
    // Explicit constructor required by Identity
    public ApplicationRole() : base() {}
    public ApplicationRole(string roleName) : base(roleName) {}
}

public interface ITenantSetter
{
    string CurrentTenantId { get; set; }
}

public class TenantService : ITenantSetter
{
    public string CurrentTenantId { get; set; }
}


public class TenantResolverMiddleware
{
    private readonly RequestDelegate _next;

    public TenantResolverMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task InvokeAsync(HttpContext context, ITenantSetter tenantSetter)
    {
        // Example Strategy: Extract from Subdomain (e.g., ://myapp.com)
        var host = context.Request.Host.Value;
        var segments = host.Split('.');
        
        string tenantId = "default";
        if (segments.Length > 2)
        {
            tenantId = segments[0].ToLower(); // returns "tenantA"
        }

        // Alternative strategy: Header fallback
        if (context.Request.Headers.TryGetValue("X-Tenant", out var headerTenant))
        {
            tenantId = headerTenant.ToString();
        }

        tenantSetter.CurrentTenantId = tenantId;
        await _next(context);
    }
}

using Microsoft.AspNetCore.Identity.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore;

public class ApplicationDbContext : IdentityDbContext<ApplicationUser, ApplicationRole, string>
{
    private readonly ITenantSetter _tenantSetter;

    public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options, ITenantSetter tenantSetter)
        : base(options)
    {
        _tenantSetter = tenantSetter;
    }

    public DbSet<Tenant> Tenants { get; set; }
    public DbSet<Product> Products { get; set; } // Example app business data

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);

        // Adjust Unique Index for Users to include Tenant ID
        builder.Entity<ApplicationUser>(entity =>
        {
            entity.HasIndex(u => new { u.NormalizedUserName, u.TenantId }).IsUnique().HasDatabaseName("UserNameIndex");
            entity.HasIndex(u => new { u.NormalizedEmail, u.TenantId }).IsUnique().HasDatabaseName("EmailIndex");
        });

        // Adjust Unique Index for Roles to include Tenant ID
        builder.Entity<ApplicationRole>(entity =>
        {
            entity.HasIndex(r => new { r.NormalizedName, r.TenantId }).IsUnique().HasDatabaseName("RoleNameIndex");
        });

        // Apply global query filter for data isolation across multi-tenant tables
        string currentTenant = _tenantSetter.CurrentTenantId;

        builder.Entity<ApplicationUser>().HasQueryFilter(u => u.TenantId == _tenantSetter.CurrentTenantId);
        builder.Entity<ApplicationRole>().HasQueryFilter(r => r.TenantId == _tenantSetter.CurrentTenantId);
        builder.Entity<Product>().HasQueryFilter(p => p.TenantId == _tenantSetter.CurrentTenantId);
    }

    // Intercept saves to auto-populate TenantId on creation
    public override int SaveChanges(bool acceptAllChangesOnSuccess)
    {
        ApplyTenantId();
        return base.SaveChanges(acceptAllChangesOnSuccess);
    }

    public override Task<int> SaveChangesAsync(bool acceptAllChangesOnSuccess, CancellationToken cancellationToken = default)
    {
        ApplyTenantId();
        return base.SaveChangesAsync(acceptAllChangesOnSuccess, cancellationToken);
    }

    private void ApplyTenantId()
    {
        var entries = ChangeTracker.Entries()
            .Where(e => e.State == EntityState.Added && e.Entity is IMustHaveTenant);

        foreach (var entry in entries)
        {
            ((IMustHaveTenant)entry.Entity).TenantId = _tenantSetter.CurrentTenantId;
        }
    }
}

using Microsoft.AspNetCore.Identity;

public class TenantAwareUserValidator : UserValidator<ApplicationUser>
{
    public override async Task ValidateAsync(UserManager<ApplicationUser> manager, ApplicationUser user)
    {
        // First run standard validations
        var result = await base.ValidateAsync(manager, user);
        if (!result.Succeeded) return result;

        // Custom validation check scoped strictly within the current tenant boundary
        var owner = await manager.FindByNameAsync(user.UserName);
        if (owner != null && owner.TenantId == user.TenantId && !string.Equals(await manager.GetUserIdAsync(owner), await manager.GetUserIdAsync(user)))
        {
            return IdentityResult.Failed(new IdentityError { Code = "DuplicateUserName", Description = $"Username '{user.UserName}' is already taken for this tenant." });
        }

        return IdentityResult.Succeeded;
    }
}

// Program 

using Microsoft.EntityFrameworkCore;
using Microsoft.AspNetCore.Identity;

var builder = WebApplication.CreateBuilder(args);

// 1. Connection string configuration
var connectionString = builder.Configuration.GetConnectionString("DefaultConnection");
builder.Services.AddDbContext<ApplicationDbContext>(options =>
    options.UseSqlServer(connectionString));

// 2. Multi-Tenancy Scope injection configuration
builder.Services.AddScoped<TenantService>();
builder.Services.AddScoped<ITenantSetter>(provider => provider.GetRequiredService<TenantService>());

// 3. Configure ASP.NET Core Identity with application variants
builder.Services.AddIdentity<ApplicationUser, ApplicationRole>(options =>
{
    options.User.RequireUniqueEmail = false; // Bypasses universal email validation block
})
.AddEntityFrameworkStores<ApplicationDbContext>()
.AddDefaultTokenProviders();

// 4. Override user validator configuration
builder.Services.AddScoped<IUserValidator<ApplicationUser>, TenantAwareUserValidator>();

builder.Services.AddControllersWithViews();

var app = builder.Build();

// 5. Middleware Placement
app.UseStaticFiles();
app.UseRouting();

// Resolve multi-tenancy state BEFORE authentication and authorization run
app.UseMiddleware<TenantResolverMiddleware>(); 

app.UseAuthentication(); // Evaluates session state
app.UseAuthorization();  // Controls access rules

app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();


// Controller example

using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.Identity;
using Microsoft.AspNetCore.Mvc;

[Authorize(Roles = "Admin")] // Secured with Role-Based Authentication
public class ProductsController : Controller
{
    private readonly ApplicationDbContext _context;
    private readonly UserManager<ApplicationUser> _userManager;

    public ProductsController(ApplicationDbContext context, UserManager<ApplicationUser> userManager)
        : _context = context;
        _userManager = userManager;
    }

    public async Task<IActionResult> Index()
    {
        // Global EF Query filters ensure only current tenant's products show up
        var products = await _context.Products.ToListAsync();
        return View(products);
    }

    [HttpPost]
    public async Task<IActionResult> Create(Product model)
    {
        if (ModelState.IsValid)
        {
            // TenantId auto-applied inside overridden DbContext SaveChanges
            _context.Products.Add(model);
            await _context.SaveChangesAsync();
            return RedirectToAction(nameof(Index));
        }
        return View(model);
    }
}
