🌐 Lens: Code for Community

This community is tring to deliver and engage engineers to practice collaborative production-grade software engineering. Our goal is to collectively design, build, and document a complete Multi-Tenant SaaS Storefront using .NET 8.0 ASP.NET Core MVC. Our focus is to encourage, engage new engineers and learners to join, learn and contribute in this community & project.

🛠️ Lens: Learning && Community

1. Real-World Problem Solving: We solve production challenges like tenant isolation pipelines, global database filtering with EF Core, and secure client-side scripting. [ON GOING]

2. Interactive Course Materials: Every major architectural component is accompanied by step-by-step documentation, real-life examples, and actionable lab challenges. [Plan]

3. Co-Authoring Code: Community members collaborate directly through GitHub Pull Requests, code reviews, and automated CI/CD pipeline building. [Plan]

4. DevOps & Delivery: We hopefully will finish by publishing our codebases to live Linux VPS servers, mastering the entire software development lifecycle (SDLC). [ON GOING]

📚 Course Curriculum Outline

Here is a structured course curriculum blueprint:
Source material designed specifically for your .NET 8.0 ASP.NET Core MVC Multi-Tenant Store application. This curriculum is architected to teach junior developers how to build a production-ready SaaS (Software as a Service) platform using Clean Architecture.

Chapter 1:

1. Architecture & SaaS Practices
2. Multi-Tenant Architecture & Domain Foundations
3. Understanding Tenant Isolation Strategies (Single DB vs. Multi-DB).
4. Setting up Clean Architecture layers (Core, Infrastructure, Web).

Chapter 2:

1. Data Isolation with EF CoreQuery Filters for automated Tenant Isolation.
2. Database migrations management on a shared platform.

Chapter 3:

1. Modern MVC with Vanilla JavaScript
2. Building dynamic, non-blocking UI storefronts.
3. Asynchronous fetching (fetch API), secure anti-forgery tokens, and modular DOM scripts.

Chapter 4:

1. Production Deployment & DevOps
2. Managing environments safely via GitHub Actions and secrets.
3. Publishing, proxy configuration (Nginx), and hosting on a Linux VPS.
