It is an excellent approach. 

You are describing an architecture known as Opaque Token Mapping or Tenant Masking.

By sending a completely random 8-digit number (the alias) to the browser instead of the real database TenantId, the browser has no idea what the true database ID or primary key is. The server acts as a secure translator.How the Masking Workflow WorksAt Login/Resolution: The system determines the true TenantId (e.g., 55). It then generates a secure 8-digit random alias (e.g., 72491563).On the Server Session: You store the mapping where the random number is the lookup key: Session.SetString("72491563", "55").To the Browser: You only pass 72491563. The true tenant database key (55) is never exposed in the HTML, cookies, or headers.Subsequent Requests: The browser sends back the random number 72491563.The Middleware: Your middleware uses 72491563 to fetch the true TenantId from the session. If found, it safely sets context.Items["CurrentTenantId"] = "55" for your database queries.Implementation Code1. Generating the Masked Mapping at LoginWhen the user logs in, generate the random number, map it to the true TenantId inside the session, and give only that random number to the client.csharp

using System.Security.Cryptography;

[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Login(LoginViewModel model)
{
    var trueTenantId = HttpContext.Items["CurrentTenantId"]?.ToString(); // e.g., "55"
    
    // ... Identity password validation logic here ...
    var result = await _signInManager.PasswordSignInAsync(model.Username, model.Password, model.RememberMe, false);
    
    if (result.Succeeded)
    {
        // 1. Generate a secure, unpredictable 8-digit random string
        int randomAlias = RandomNumberGenerator.GetInt32(10000000, 100000000); 
        string tenantMask = randomAlias.ToString(); // e.g., "72491563"

        // 2. Store the mapping in the Server Session (Key = Mask, Value = True ID)
        HttpContext.Session.SetString($"TenantMapping_{tenantMask}", trueTenantId);
        
        // Also save what mask this specific active login session is authorized to use
        HttpContext.Session.SetString("ActiveTenantMask", tenantMask);

        // 3. Send ONLY the mask to the browser (via custom header or secure cookie)
        Response.Cookies.Append("X-Tenant-Mask", tenantMask, new CookieOptions 
        { 
            Secure = true, 
            SameSite = SameSiteMode.Strict,
            HttpOnly = false // Allow JS to read if needed, or set to true for absolute safety
        });

        return RedirectToAction("Index", "Dashboard");
    }

    return View(model);
}

2. Translating the Mask back to True ID in MiddlewareYour middleware intercepts incoming requests, reads the public 8-digit mask, translates it back to your internal database key, and rejects the request if the mask is fake or altered.csharp

public class TenantResolverMiddleware
{
    private readonly RequestDelegate _next;

    public TenantResolverMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        // 1. Extract the 8-digit random mask from the header or cookie
        string incomingMask = context.Request.Headers["X-Tenant-Mask"].ToString();
        if (string.IsNullOrEmpty(incomingMask))
        {
            incomingMask = context.Request.Cookies["X-Tenant-Mask"];
        }

        // 2. If the user is authenticated, enforce strict validation
        if (context.User.Identity?.IsAuthenticated == true)
        {
            // Ensure the incoming mask is the exact one authorized during login
            var authorizedMask = context.Session.GetString("ActiveTenantMask");

            if (string.IsNullOrEmpty(incomingMask) || incomingMask != authorizedMask)
            {
                context.Response.StatusCode = StatusCodes.Status403Forbidden;
                await context.Response.WriteAsync("Security Violation: Invalid or tampered tenant token.");
                return;
            }

            // 3. Translate the mask back to the TRUE internal Database TenantId
            var trueTenantId = context.Session.GetString($"TenantMapping_{incomingMask}");

            if (string.IsNullOrEmpty(trueTenantId))
            {
                context.Response.StatusCode = StatusCodes.Status403Forbidden;
                await context.Response.WriteAsync("Security Violation: Expired or unrecognized tenant session.");
                return;
            }

            // 4. Inject the TRUE ID silently into context items for EF Core usage
            // Downstream code and databases see "55", but the browser only ever saw "72491563"
            context.Items["CurrentTenantId"] = trueTenantId;
        }
        else
        {
            // For anonymous pages before login, resolve tenant by URL/Subdomain normally
            context.Items["CurrentTenantId"] = ParseTenantFromUrl(context.Request.Host.Host);
        }

        await _next(context);
    }
}

To use this Opaque Tenant Masking strategy during the initial tenant resolution phase in your middleware (before the user logs in), you must change how you look up the tenant from the URL.When an anonymous user visits a URL (e.g., ://myapp.com or ://myapp.com), the middleware generates the random 8-digit mask right then, saves it to the session, and uses it going forward.

Step 1: The Multi-Strategy Resolution Middleware

This middleware handles two paths:Anonymous / 

First Request: Looks up the true TenantId by URL, generates the 8-digit mask, maps it in the Session, and gives it to the browser via a cookie.Subsequent Requests: Reads the 8-digit mask from the cookie or header, verifies it against the session, and translates it back to the true TenantId.

csharp

using Microsoft.AspNetCore.Http;
using System.Security.Cryptography;
using System.Threading.Tasks;

public class TenantResolverMiddleware
{
    private readonly RequestDelegate _next;

    public TenantResolverMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task InvokeAsync(HttpContext context, ITenantLookupService tenantLookupService)
    {
        string trueTenantId = null;

        // 1. Try to get the 8-digit random mask from the request
        string incomingMask = context.Request.Cookies["X-Tenant-Mask"];
        if (string.IsNullOrEmpty(incomingMask) && context.Request.Headers.TryGetValue("X-Tenant-Mask", out var headerMask))
        {
            incomingMask = headerMask;
        }

        // 2. STRATEGY A: The browser already has an 8-digit mask
        if (!string.IsNullOrEmpty(incomingMask))
        {
            // Translate the mask back to the true internal Database TenantId using the Session
            trueTenantId = context.Session.GetString($"TenantMapping_{incomingMask}");

            // Double check: If the user is logged in, ensure this is their authorized mask
            if (context.User.Identity?.IsAuthenticated == true)
            {
                var authorizedMask = context.Session.GetString("ActiveTenantMask");
                if (incomingMask != authorizedMask || string.IsNullOrEmpty(trueTenantId))
                {
                    context.Response.StatusCode = StatusCodes.Status403Forbidden;
                    await context.Response.WriteAsync("Security Violation: Invalid or tampered tenant mask.");
                    return;
                }
            }
        }

        // 3. STRATEGY B: First time visit / Mask missing or expired in Session
        if (string.IsNullOrEmpty(trueTenantId))
        {
            // Resolve the true tenant from the URL (Subdomain, Domain, or Path)
            string urlIdentifier = context.Request.Host.Host; // e.g., "://myapp.com"
            trueTenantId = await tenantLookupService.GetTrueTenantIdFromUrlAsync(urlIdentifier);

            if (string.IsNullOrEmpty(trueTenantId))
            {
                context.Response.StatusCode = StatusCodes.Status404NotFound;
                await context.Response.WriteAsync("Valid tenant domain could not be resolved.");
                return;
            }

            // Generate a secure, unique 8-digit random mask
            int randomAlias = RandomNumberGenerator.GetInt32(10000000, 100000000);
            incomingMask = randomAlias.ToString();

            // Store the mapping securely in the Server Session (Key = Mask, Value = True ID)
            context.Session.SetString($"TenantMapping_{incomingMask}", trueTenantId);

            // Send the mask to the browser so subsequent requests include it
            var cookieOptions = new CookieOptions
            {
                Secure = true,
                SameSite = SameSiteMode.Strict,
                HttpOnly = false, // Set to true if you do not need Javascript to read it
                IsEssential = true // Ensures cookie works even if user rejects tracking cookies
            };
            context.Response.Cookies.Append("X-Tenant-Mask", incomingMask, cookieOptions);
        }

        // 4. Success: Inject the TRUE database ID silently into the request pipeline
        // Your database, EF Core, and business logic only ever see the true ID (e.g. "55")
        context.Items["CurrentTenantId"] = trueTenantId;

        await _next(context);
    }
}

[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Login(LoginViewModel model)
{
    // The middleware already translated the 8-digit mask to the true database ID for us!
    var trueTenantId = HttpContext.Items["CurrentTenantId"]?.ToString();

    var user = await _userManager.FindByNameAsync(model.Username);
    if (user == null || user.TenantId != trueTenantId)
    {
        ModelState.AddModelError("", "Invalid login attempt for this tenant.");
        return View(model);
    }

    var result = await _signInManager.PasswordSignInAsync(user, model.Password, model.RememberMe, false);
    if (result.Succeeded)
    {
        // Lock the specific mask they are using as the authorized mask for this login session
        string currentMask = Request.Cookies["X-Tenant-Mask"];
        HttpContext.Session.SetString("ActiveTenantMask", currentMask);

        return RedirectToAction("Index", "Dashboard");
    }

    ModelState.AddModelError("", "Invalid login attempt.");
    return View(model);
}
