A navigation bar (navbar) is a structural user interface element that contains hyperlinks to guide users through different sections or pages of a website.
 
Why and Where Navigation Bars Are Used ad Why Used: 
They improve website usability, lower bounce rates, and boost search engine optimization (SEO) by signaling site hierarchy to search engine crawlers.
 
Where Used: 
They are traditionally placed within the <header> element at the very top of a webpage,.Secondary navbars frequently appear in sidebars or deep within page footers.
 
Core Structure: The HTML Backbone
The industry standard builds a navbar out of semantic, nested HTML5 elements.

<nav>: Tells screen readers and browsers this is a major navigation section.
<ul>: An unordered list grouping the menu choices.
<li>: Wraps individual link nodes for robust layout control.
<a>: The interactive hyperlink mapping the target URL.

html<!-- Base Semantic HTML Template -->

<nav class="navbar">
  <ul class="nav-links">
    <li><a href="#home">Home</a></li>
    <li><a href="#services">Services</a></li>
    <li><a href="#about">About</a></li>
  </ul>
</nav>

How to Make It Dynamic:

A static HTML menu requires hardcoding items page-by-page. A dynamic navbar adapts on-the-fly based on state changes (like 
1. screen resizing, 
2. page scrolling, 
3. user login state, 
4. or active location tracking).

### Dynamic Example 1: 

Highlighting the Active Page (JavaScript UI State)
This script examines the user's browser location URL and automatically highlights the menu tab that matches, signaling where they are on the site map.

javascript// Run script after document DOM trees render completely

document.addEventListener("DOMContentLoaded", () => {
  const currentUrl = window.location.href;
  const navLinks = document.querySelectorAll(".nav-links a");

  navLinks.forEach(link => {
    // If the link address matches the active window window location
    if (link.href === currentUrl) {
      link.classList.add("active"); // Apply specific styling rule
      link.setAttribute("aria-current", "page"); // Accessibility support
    }
  });
});

To change the navigation layout dynamically based on a user's choice across a multi-page website, you need to solve one specific problem: 

1. persisting the layout choice when clicking to a new page. Because standard HTML pages wipe out JavaScript state upon reloading, we must save the choice to the browser's storage (localStorage).

Here is a complete, modular solution using Vanilla JavaScript and clean semantic markup.

The Vanilla JavaScript Layout Engine (nav-engine.js)This script renders the HTML content, parses your current browser address string to accurately auto-highlight active state links, and manages cross-page user choices instantly via localStorage.javascriptdocument.addEventListener("DOMContentLoaded", () => {
  const navContainer = document.getElementById("navbar-container");
  const layoutSelector = document.getElementById("layoutSelector");

  // 1. Centralized navigation data array 
  const menuItems = [
    { text: "Home", href: "index.html" },
    { text: "Services", href: "services.html" },
    { text: "About Us", href: "about.html" },
    { text: "Contact", href: "contact.html" }
  ];

  // 2. Generate and inject HTML dynamically
  function renderNavbar() {
    const currentFile = window.location.pathname.split("/").pop() || "index.html";
    
    let linksHtml = menuItems.map(item => {
      // Check if item match current window page file
      const isActive = currentFile === item.href ? 'class="active" aria-current="page"' : '';
      return `<li><a href="${item.href}" ${isActive}>${item.text}</a></li>`;
    }).join("");

    navContainer.innerHTML = `
      <nav class="navbar">
        <ul class="nav-links">
          ${linksHtml}
        </ul>
      </nav>
    `;
  }

  // 3. Dynamic Application of Selection Style
  function applyLayout(chosenLayout) {
    // Inject attribute string directly onto body node to trigger specialized CSS blocks
    document.body.setAttribute("data-nav-layout", chosenLayout);
    // Remember layout configuration value across subsequent system loads
    localStorage.setItem("userNavLayout", chosenLayout);
    // Sync the dropdown menu value visual state
    if (layoutSelector) layoutSelector.value = chosenLayout;
  }

  // 4. Initialization Logic
  renderNavbar();
  
  // Fetch previously stored layout choice, default to 'horizontal' layout if empty
  const savedLayout = localStorage.getItem("userNavLayout") || "horizontal";
  applyLayout(savedLayout);

  // 5. Change handler event loop listener
  if (layoutSelector) {
    layoutSelector.addEventListener("change", (e) => {
      applyLayout(e.target.value);
    });
  }
});
