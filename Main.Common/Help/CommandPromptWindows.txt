4. For .Net 10.0 which is in long term spport program and will continue for future releases into a single 
   name and platform independent. So, The Model classes are free from the backward compatibility issues.
5. Seperation of concern is achieved with such design. [EF, Data annotaton is the concern here (which should 
   not be with POCO classes)]; actual concern goes to the Data project which are related to database logics.
